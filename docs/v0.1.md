# v0.1 发布说明

> 这是 `v0.1.0` 的历史发布说明，不代表当前 `master`。Codex OAuth 等后续能力见 [v0.2 发布说明](v0.2.md)。

v0.1 的目标是让 LLM API Factory 成为一个可本地部署、可远程代理、可观测的个人 LLM API 网关。

## 已完成

- OpenAI / Anthropic / Gemini 标准入口
- 标准 provider 最小侵入代理
- Custom provider 强定制能力
- Factory key 与规则组权限隔离
- 顺序主备、weighted round robin、fallback 和熔断
- API endpoint/key 管理
- 单 key 直测，支持 chat / responses / Codex / Claude Messages / Claude Code / Gemini 模板
- Agent 创建、部署命令、心跳、WebSocket 代理、禁用/排空/删除
- API key 测试经 Agent 发起
- 请求日志、尝试日志、route explain
- token、cached tokens、cache hit、stream complete
- dump 文件和 dump index
- Redis 运行态、SQLite 默认存储、PostgreSQL 双模式支持
- 管理台登录，不再把 admin token 打进前端 bundle
- 上游密钥加密、Factory key 哈希存储
- Agent SSRF 目标校验

## 推荐发布姿势

1. `master` 作为 v0.1 发布分支。
2. 打 tag：

```bash
git tag v0.1.0
git push origin v0.1.0
```

3. 生产部署 Agent 时，`--repo-ref` 使用 `v0.1.0`，避免追随主分支变化。

```bash
curl -fsSL https://raw.githubusercontent.com/Kuaizr/llm_api_factory/v0.1.0/scripts/agent_install.sh | bash -s -- \
  --repo-ref v0.1.0 \
  --ws-url ws://factory.example.com/agent/ws \
  --heartbeat-url http://factory.example.com/agent/heartbeat \
  --agent-name edge-hk \
  --agent-token your-agent-token
```

## v0.1 边界

- 不做公共多租户计费平台。
- 不做跨 provider 协议翻译。
- 不内置订阅账号 OAuth 转 API。
- SQLite 可用，但高并发建议 PostgreSQL。
- 多 worker 扩容前需要继续收敛运行态到 Redis。
